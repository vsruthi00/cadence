# Prompt: session breakdown

Used by flow/plan.md after writing-plans produces the implementation plan.

1. Dispatch council scoped to the executor role ONLY (not the full bench): hand it the finished implementation plan and ask for a session breakdown.
2. The executor returns, per its session-planning responsibility: sessions (tasks grouped), a context-estimate band per session (light / moderate / heavy with a one-line justification), split points (safe-to-close boundaries), and backup points (earlier safe-stops if scope creeps).
3. Write the breakdown into the master plan's Session Breakdown section (schema/master-plan.template.md).
4. State the honest limit in the plan: the estimate is coarse; the live statusline is the real number, and the estimate is corrected against actuals each session.

This is a scoped, cheap call. Do not convene other roles for the breakdown.
